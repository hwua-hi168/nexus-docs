# AI Playground

## 概述

Hi168 平台的 **AI Playground** 模块提供统一的大语言模型对话、图像生成、视频生成体验。用户可以在同一界面中切换不同供应商的 AI 模型，上传图片或文件作为上下文参考，进行多轮对话、内容创作和代码编写。

---

## 一、功能说明

### 1.1 核心特性

| 特性 | 说明 |
| --- | --- |
| **多模型切换** | 支持 OpenAI、Anthropic、Google Gemini、xAI 等多家供应商的模型，一键切换 |
| **多轮对话** | 自动维护上下文，支持设置上下文轮数 |
| **图像生成** | 选择图像模型（如 DALL-E、gpt-image-1）即可通过文字描述生成图像 |
| **视频生成** | 选择视频模型（如 Sora 2、Sora 2 Pro）即可通过文字描述生成视频 |
| **图片参考** | 上传图片作为视频生成的参考素材，引导生成风格和内容 |
| **文件上传** | 上传文本文件作为对话上下文，模型可基于文件内容回答问题 |
| **对话管理** | 支持多会话管理、历史记录、对话导出 |
| **参数调节** | 可自定义 Temperature、Top-P、最大 Token 数等模型参数 |

## 二、附件上传说明

Playground 支持两种附件上传方式：**图片上传** 和 **文件上传**，单个文件大小限制为 **5MB**。

### 2.1 图片上传

点击输入框左侧的 **图片按钮**（📷）上传图片，或直接 **粘贴剪贴板中的图片**。

**支持的图片格式：**

| 格式 | 扩展名 | 说明 |
| --- | --- | --- |
| JPEG | `.jpg` `.jpeg` | 通用照片格式 |
| PNG | `.png` | 支持透明背景 |
| GIF | `.gif` | 静态或动态图片 |
| WebP | `.webp` | 高压缩率的现代图片格式 |

**图片用途：**
- **对话模型**：图片作为视觉上下文，模型可分析图片内容并回答相关问题
- **视频模型**：图片作为参考素材（`input_reference`），引导 Sora 生成与图片风格一致的视频

### 2.2 文件上传

点击输入框左侧的 **文件按钮**（📎）上传文本文件。文件内容会作为对话上下文发送给模型。

**支持的文件格式：**

| 分类 | 扩展名 | 说明 |
| --- | --- | --- |
| **文本** | `.txt` | 纯文本文件 |
| **文档** | `.md` | Markdown 文档 |
| **数据** | `.json` `.csv` `.xml` `.yaml` `.yml` | 结构化数据文件 |
| **代码** | `.py` `.js` `.ts` `.go` `.java` `.c` `.cpp` `.h` `.html` `.css` `.sql` | 源代码文件 |
| **脚本/配置** | `.sh` `.bat` `.ini` `.cfg` `.conf` `.toml` `.env` `.log` | 脚本和配置文件 |

> **注意**：不支持 PDF、Word（`.docx`）、Excel（`.xlsx`）等二进制文档格式。如需使用这类文件内容，请先将其转换为纯文本或 Markdown 格式后上传。

### 2.3 上传限制

| 限制项 | 值 |
| --- | --- |
| 单个文件大小 | ≤ 5MB |
| 图片上传方式 | 点击上传、拖拽粘贴 |
| 文件上传方式 | 点击上传 |
| 支持多选 | 是 |

## 三、模型分类

### 3.1 对话模型（Chat）

用于多轮对话、问答、代码编写、文本翻译等。支持上传图片和文件作为上下文。

### 3.2 图像模型（Image）

用于根据文字描述生成图片。选择图像类别的模型后，输入描述即可生成。

### 3.3 视频模型（Video）

用于根据文字描述生成视频。可上传图片或视频作为参考素材引导视频风格。

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 分辨率 | 1280×720 | 横版高清 |
| 时长 | 8 秒 | 可选 4/8/12 秒（可在提示词中直接口述，如“生成12秒宣传片”）|
| 参考图片/视频 | 可选 | 上传图片或视频作为参考素材，仅支持单个文件 |

> **注意**：Sora/Sora2 Pro 目前每次仅支持生成一个完整视频片段（最长12秒），不支持分镜头或多场景拼接。建议将所有内容简明描述在一个提示词中。

#### Seadance 说明

Seadance 也是“一次生成一个完整视频片段”，不支持自动分场景或多镜头拼接。每次生成需单独描述完整内容。

## 四、API 令牌

使用 Playground 前需要创建 API 令牌（`sk-nxl-` 开头）。进入「API 令牌」页面创建令牌后，在 Playground 参数设置中填入即可。

令牌也可用于通过 OpenAI 兼容 API 从外部应用（如 Cursor、Continue、VS Code Copilot）接入 Hi168 的模型服务。

### 4.1 令牌安全须知

- API 令牌（`sk-nxl-...`）是您访问 AI 服务的凭证，**请妥善保管，不要泄露**
- 令牌创建后仅展示一次完整密钥，后续只能看到前缀
- 如果密钥泄露，请立即在「API 令牌」页面删除并重新创建
- 建议为不同的应用场景创建独立的令牌，方便管理和追踪用量

## 五、虚拟 Token（VToken）计算

不同模型价格差异大，平台用 **VToken** 统一换算额度——便宜模型消耗少、贵的模型消耗多。

**计算公式：**

> 消耗 VToken = 输入 Token × 输入系数 + 输出 Token × 输出系数

**举例：** 一次对话用了 1,000 输入 + 500 输出 Token

| 模型 | 输入系数 | 输出系数 | 消耗 VToken |
| --- | --- | --- | --- |
| Claude Haiku 4.5（经济型） | 0.06 | 0.28 | 1000×0.06 + 500×0.28 = **200** |
| Claude Opus 4（高端） | 1.03 | 5.12 | 1000×1.03 + 500×5.12 = **3,590** |

> 各模型的消耗系数可在「**模型广场**」中查看。

## 六、扣费流程

```
用户发起请求 → ① 前置检查 → ② 调用模型 → ③ 成功后扣费
                  ↓ 不通过          ↓ 失败
              返回错误，不扣费    记录日志，不扣费
```

**① 前置检查（不通过则不扣费）：**
- 是否有生效中的订阅
- 额度或时间是否已用尽
- 是否超过速率限制（RPM / TPM）

**② 成功后扣费：**
1. 统计本次输入/输出 Token
2. 按模型消耗系数折算为 VToken
3. 从套餐额度中扣除

> 模型返回错误（网络超时、服务异常等）**不扣费**，但会记录日志。

---

## 七、速率限制（Rate Limit）

为保障服务质量，不同套餐有不同的速率限制：

| 限制类型 | 说明 |
| --- | --- |
| **RPM**（每分钟请求数） | 由套餐等级决定，详情看套餐 |
| **TPM**（每分钟 Token 数） | 由 API 令牌独立设置，默认 50 万 TPM |

超过限制时，API 会返回 `429 Too Many Requests`，稍等后重试即可。

---

## 八、额度用完怎么办

| 场景 | 系统行为 | 建议 |
| --- | --- | --- |
| 额度用完 | 订阅标记"已过期"，API 报错 | 前往「套餐订阅」续费 |
| 时间到期 | 同上 | 购买新套餐 |
| 频率超限 | 返回 `429` | 降低频率或升级套餐 |

> 套餐到期后，API 令牌和历史记录**不会删除**，续费即可继续使用。

## 九、常见问题

| 问题 | 回答 |
| --- | --- |
| 调用返回"没有有效订阅"？ | 未订阅或已过期，前往「套餐订阅」购买 |
| 不同模型费用一样吗？ | 不一样，各模型消耗系数不同，详见「模型广场」 |
| 调用失败会扣费吗？ | 不会，只有模型成功返回才扣费 |
| 套餐到期数据还在吗？ | 在，令牌和记录都保留，续费后继续使用 |
| 能同时有多个套餐吗？ | 不能，新订阅自动替换旧订阅 |
| 怎么查用量？ | 「数据看板」看统计，「使用日志」看明细，「套餐订阅」看剩余额度 |
